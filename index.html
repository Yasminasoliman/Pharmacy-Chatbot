Pipeline Nodes
What Each Node Does
🖼️
ocr_node
Extracts medicine names from prescription images. Tries vision LLM (GPT-4o/Gemini), falls back to Tesseract. Reformulates a clean text query for downstream nodes.
🧠
intent_node
Classifies query as need_search, answer_directly, or contact_support using structured Pydantic LLM output. Routes the graph to the correct branch.
🎫
support_node NEW
Detects support sub-intent, creates a structured RX-XXXXXXXX ticket, sends it via SMTP and/or REST API, and returns a human-friendly acknowledgement. Emergency → instant escalation.
🔍
search_node + tool_node
Invokes the right pharmacy tool (medicine lookup, disease lookup, generic name, drug info). Loops via ToolNode until all needed data is retrieved.
📝
answer_node
Synthesises tool results into a clear, human-friendly answer. Passes the response to the pharmacy availability agent for stock enrichment.
🏪
pharmacy_agent_node v2
Checks real-time stock for every medicine in the answer. Connects to PHARMACY_API_URL or uses a realistic Egyptian medicine mock dataset as fallback.
🛡️
safety_node
Reviews all paths — drug answers and support responses — for dangerous dosage advice or harmful content before delivering to the user.
📊
eval_node v2
LLM-judge scores every response on relevance, safety, accuracy, and completeness (≥6/10 pass). Results logged to eval_logs/eval_results.jsonl.
🔗
MCP Server v2
Exposes all 5 pharmacy tools via Model Context Protocol. Copy one config block to Claude Desktop — no API integration needed, SSE-based.
AI Tooling
5 Pharmacy Intelligence Tools
Each tool queries a PostgreSQL database combining SQL fuzzy search (pg_trgm) with semantic vector search (pgvector + BGE embeddings).
Tool Input Returns Search Example Query
medicine_lookup Medicine name Price, generic name, disease, prescription status, manufacturer SQL ILIKE "What is Panadol?"
disease_lookup Disease / condition List of medicines that treat the condition Fuzzy SQL "What treats migraines?"
generic_name_lookup Active ingredient All brand names for that ingredient SQL ILIKE "Brands of Ibuprofen?"
drug_information Medicine name Mechanism, side effects, contraindications Vector (pgvector) "How does Metformin work?"
check_pharmacy_stock Medicine name In-stock status, quantity, price, prescription note API / Mock Real-time availability
Database Stack
PostgreSQL 16 via Docker · pgvector for semantic search · pg_trgm for fuzzy matching · BGE-small-en-v1.5 embeddings ingested at setup
Hybrid Search
Brand/generic lookups use fast ILIKE SQL. Mechanism and side-effect queries use semantic vector similarity for natural language understanding.
Quality Assurance
Automatic 4-Dimension LLM Evaluation
Every response — drug answers and support replies — is scored by an LLM judge before delivery. Results queryable via API.
Relevance
≥ 6 / 10 pass
Does the answer actually address what the user asked?
Safety
≥ 6 / 10 pass
No dangerous dosage guidance or harmful drug interaction advice?
Accuracy
≥ 6 / 10 pass
Consistent with the data returned by the pharmacy tools?
Completeness
≥ 6 / 10 pass
Are all key data points from tool results covered in the answer?

Technology
Full-Stack Tech Stack

AI / Agents
  • LangGraph — stateful multi-agent orchestration
